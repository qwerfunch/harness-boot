# harness-boot

**Boot a detailed plan MD into a Claude Code native multi-agent harness.**

> If you already have a detailed plan MD, `harness-boot` reads it and automatically generates an **executable harness**. Built-in: TDD sub-agent isolation, code-doc sync runtime enforcement, anti-rationalization skills, and Opus/Sonnet model routing.

## Key Features

- **Plan-to-harness auto-conversion** — Generates 50+ file structure (9 agents, 8 skills, 5 hooks, 5 protocols) in one go
- **TDD sub-agent context isolation** — Separates Red/Green/Refactor into distinct sub-agents
- **Runtime guardrails (hooks.json)** — Enforces security, auto-formatting, and doc sync at the system level
- **Anti-rationalization** — Every skill embeds agent excuse-rebuttal tables
- **Model routing** — Opus (judgment) / Sonnet (execution) split for ~30% cost reduction
- **Cross-session state management** — Automatic Initializer/Coding mode switching with checkpoint/resume and state consistency verification
- **Quality gates with evidence** — 5-stage gates (TDD → Implementation → Review → Testing → Deploy), each requiring concrete evidence to pass

## Installation

```bash
git clone <this-repo>
cd harness-boot
claude --plugin-dir .
```

## Usage

```
1. Write a detailed plan MD (project requirements, feature specs, etc.)

2. Boot the harness in Claude Code:
   /setup path/to/plan.md

3. Generate Phases 1-6, confirm each phase before proceeding

4. Start development:
   /start
```

### Commands

| Command | Purpose | When to Use |
|---------|---------|-------------|
| `/setup <plan.md>` | Generate harness structure from a detailed plan | Once at project start |
| `/start` | Start development (run TDD on next feature) | After harness is ready, use repeatedly |

## Structure Generated by `/setup` (in target project)

```
project-root/
├── CLAUDE.md                  # Main summary (<1,500 tokens)
├── PROGRESS.md                # State tracking
├── feature-list.json          # Feature list + pass status
├── CHANGELOG.md               # Harness change history
├── .claude/
│   ├── settings.json          # Hook configuration
│   ├── agents/                # 9 sub-agents
│   ├── skills/                # 8 skills (Anthropic Agent Skills format, 7-section anatomy)
│   ├── protocols/             # TDD, iteration convergence, doc sync, etc.
│   ├── examples/              # Golden samples
│   └── *.md                   # context-map, security, quality-gates, etc.
├── hooks/                     # 5 executable hook scripts
└── scripts/                   # Automation utilities
```

## Four Design Principles

1. **TDD-First** — Context isolation via 3-way sub-agent split
2. **Iteration Convergence** — Max 5 loops, escalate if exceeded
3. **Code-Doc Sync** — Triple defense (prompt + PreToolUse hook + reviewer)
4. **Anti-Rationalization** — Excuse-rebuttal tables embedded in every skill
